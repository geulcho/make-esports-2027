# PRD — MAKE Esports Combat Engine

## 개요
각 세트(게임)는 3개 페이즈(Early / Mid / Late)로 구성되며, 팀의 스탯과 현재 메타 환경을 바탕으로 전투(Fight)를 반복해 모멘텀을 쌓는 방식으로 승패가 결정된다. 세트는 Bo3 시리즈로 묶여 최종 시리즈 승자가 결정된다.

---

## 1. 팀 스탯

| 스탯 | 풀 네임 | 역할 |
|------|---------|------|
| MAC | Macro | 조합 주도권 (Early 전투 횟수 결정, Late 전투 핵심) |
| DRF | Draft | 메타 적응력 (CP 기반값 결정) |
| TMF | Teamfight | 집단전 능력 (Mid 전투 핵심) |
| LAN | Laning | 라인전 (Early 전투 핵심) |
| MEC | Mechanics | 기계적 실력 (전 페이즈 보조) |

- 값 범위: 50 ~ 99

---

## 2. 메타 시스템

- 7개 조합 슬롯 존재 (A, B, C, D, E, F, G)
- 매 시리즈마다 2개 조합이 활성 메타로 랜덤 선정
- 각 팀은 선호 조합(preferredCombos) 목록을 보유

### 메타 적중 여부 → DRF 효율 보정

| 조건 | DRF 보정 |
|------|----------|
| 선호 조합 ∩ 활성 메타 ≠ ∅ | +10 |
| 미적중 | −5 |

```
effDRF = clamp(DRF + 보정, 20, 100)
BaseCP  = effDRF × 1.5 + rand(1, 5)
```

---

## 3. 세트 진행 구조

### Phase 1 — Early Game (LAN + MEC 중심)

- 전투 횟수: MAC 우위 팀이 CP도 높으면 2회, 아니면 1회
- 전투 파워: `CP + LAN + MEC + rand(1, 50)`
- 보상: 전투 승자 +rand(2, 3) 모멘텀, CP

### Phase 2 — Mid Game (TMF + MEC 중심)

- 전투 횟수: rand(2, 4)
- 전투 파워: `CP + TMF + MEC + rand(1, 50)`
- 보상: 전투 승자 +rand(1, 2) 모멘텀, CP
- **콜드 게임**: 모멘텀 격차 ≥ 10이면 Mid에서 즉시 종료

### Phase 3 — Late Game (MAC + MEC 중심)

- 전투 횟수: rand(2, 3)
- 전투 파워: `CP + MAC + MEC + rand(1, 50)`
- 보상: 전투 승자 +rand(1, 2) 모멘텀, CP
- 종료 조건 (우선순위 순):
  1. 모멘텀 격차 ≥ 7 → 넥서스 파괴
  2. 어느 팀이든 모멘텀 ≥ 15 → 임계치 달성
  3. 전투 소진 → 판정승

---

## 4. 전투 해결 — Power² 확률 시스템

```
P(A 승) = pA² / (pA² + pB²)
```

| 파워 비율 | 약팀 승률 |
|-----------|-----------|
| 1.0× (동등) | 50% |
| 1.1× | ~45% |
| 1.5× | ~31% |
| 2.0× | ~20% |

강팀이 유리하나 언제나 역전 가능. ELO 600 차이 수준에서도 약팀 약 32% 전투 승리 기대.

---

## 5. 세트 승패 결정

```
모멘텀 A > 모멘텀 B → A 승
```

---

## 6. Bo3 시리즈

- 2세트 선취 팀이 시리즈 승리
- 최대 3세트 진행
- 각 세트의 최종 모멘텀 값이 기록됨 (통계용)

---

## 7. ELO 변동

- 시리즈 단위로 1회만 계산 (세트별 계산 없음)
- K값: 32
- 공식: `ΔElo = K × (결과 − 기댓값)`
  - 기댓값 = `1 / (1 + 10^((eloB - eloA) / 400))`
- 승자 +Δ, 패자 −Δ (대칭)

---

## 8. 배당률

- ELO 기반 십진법 배당
- 하우스 엣지 5%

```
pA = 1 / (1 + 10^((eloB - eloA) / 400))
oddsA = 0.95 / pA   (최소 1.01)
oddsB = 0.95 / pB   (최소 1.01)
```
