# PRD — MAKE Esports Simulator Progression

## 개요
시뮬레이터는 52주 단위 연간 사이클로 운영된다. 사용자가 "Simulate Phase" 버튼을 누를 때마다 한 서브페이즈씩 전진하며, 각 서브페이즈는 해당 시기에 진행 중인 대회의 경기들을 일괄 처리한다.

## 2. 서브페이즈 (시뮬레이션 최소 단위)

- 사용자 1클릭 = 1 서브페이즈 진행
- 지역 리그: 스플릿당 **4 서브페이즈** (각 1주치 경기량)
- 국제대회 / RTM / 플레이오프: 매치데이 단위 (구조별 상이)
- 서브페이즈 번호는 전체 시즌 통산 연속 증가

### 4-서브페이즈 블록 표시
```
현재 서브페이즈 P → 블록 번호: ceil(P / 4), 블록 내 위치: ((P-1) % 4) + 1
```

## 3. 지역 리그 시뮬레이션

### 리그 규모별 페이즈 배분 (스플릿 1회 = SRR)

| 팀 수 | 총 라운드 | Phase 1 | Phase 2 | Phase 3 | Phase 4 |
|-------|-----------|---------|---------|---------|---------|
| 8팀 | 7 | 2라운드 | 1.5라운드 | 1.5라운드 | 2라운드 |
| 10팀 | 9 | 2라운드 | 2.5라운드 | 2라운드 | 2.5라운드 |
| 12팀 | 11 | 2라운드 | 3라운드 | 3라운드 | 3라운드 |

> 실제 배분은 일별 경기 스케줄에 따라 리그마다 다름 (league_format_*.md 참조)

### 스케줄 생성
- 서클 메서드(Circle Method)로 SRR 라운드 생성
- 스케줄은 매 페이즈 시뮬레이션 시 온디맨드 계산 (localStorage 미저장)

### 순위 결정 (타이브레이커 순)
1. 시리즈 승수 (W)
2. 세트 득실 (setsFor − setsAgainst)
3. 스코어 득실 (momFor − momAgainst)

### 누적 성적
- 스플릿 1~4의 성적이 TeamRecord에 계속 누적됨
- RTM 진출권: 스플릿 1~2 누적 상위 N팀 (리그별 상이)
- 플레이오프 진출권: 스플릿 1~4 누적 상위 N팀 (리그별 상이)

## 5. 상태 관리 (Zustand Store)

### 핵심 필드
| 필드 | 타입 | 설명 |
|------|------|------|
| currentPhase | number | 현재 서브페이즈 번호 (1부터 연속 증가) |
| gameDate | string | 현재 게임 내 날짜 (YYYY-MM-DD) |
| activeMeta | string[] | 현재 활성 메타 조합 2개 |
| leagueStates | Record<leagueId, LeagueSimState> | 리그별 순위/결과 |

### LeagueSimState
```ts
{
  leagueId: string
  standings: TeamRecord[]   // 누적 성적, ELO 추적
  results: PhaseResult[]    // 완료된 페이즈별 경기 로그
}
```

### 영속성
- Zustand `persist` 미들웨어로 localStorage 저장
- 스토어 키: `make-esports-store-v3`
- 스케줄은 저장하지 않음 (온디맨드 재계산)
